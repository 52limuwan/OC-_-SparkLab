# 管理员课程管理功能

## 功能概述

管理员可以完整管理系统中的课程和实验，包括创建、编辑、删除、发布等操作。

## 主要功能

### 1. 课程管理

#### 课程列表
- 显示所有课程（包括已发布和未发布）
- 每个课程卡片显示：
  - 课程标题
  - 发布状态标签（已发布/未发布）
  - 课程描述
  - 实验数量
  - 难度等级
  - 预计时长

#### 课程操作
- **新建课程**：创建新的课程
- **编辑课程**：修改课程信息
- **删除课程**：删除课程及其下所有实验
- **发布/取消发布**：控制课程对学生的可见性
- **管理实验**：进入实验管理界面

#### 课程表单字段
- 课程名称 *（必填）
- 课程描述 *（必填）
- 难度等级 *（入门/进阶/高级）
- 预计时长 *（分钟）
- 封面图片 URL（可选）
- 发布状态（已发布/未发布）

### 2. 实验管理

#### 实验列表
- 显示选定课程下的所有实验
- 按排序号显示
- 每个实验显示：
  - 排序号
  - 实验标题
  - 实验描述
  - 难度等级
  - 时间限制
  - 分数

#### 实验操作
- **新建实验**：为课程添加新实验
- **编辑实验**：修改实验内容
- **删除实验**：删除指定实验

#### 实验表单字段

**基本信息**
- 实验名称 *（必填）
- 排序 *（必填，数字）
- 实验描述 *（必填）
- 实验内容 *（必填，Markdown 格式）

**难度和评分**
- 难度 *（入门/进阶/高级）
- 时限 *（分钟）
- 分数 *（整数）

**容器配置**
- Docker 镜像 *（例如：ubuntu:22.04）
- CPU 限制 *（0.1-4 核）
- 内存限制 *（MB，最小 128）

## 使用流程

### 创建完整课程

1. **创建课程**
   - 点击"新建课程"按钮
   - 填写课程基本信息
   - 选择"未发布"状态（先不公开）
   - 保存课程

2. **添加实验**
   - 在课程卡片上点击"管理实验"
   - 点击"新建实验"
   - 填写实验信息：
     - 基本信息（名称、描述）
     - 实验内容（Markdown 格式）
     - 难度和评分设置
     - 容器配置
   - 保存实验
   - 重复添加多个实验

3. **发布课程**
   - 返回课程列表
   - 点击课程的"发布"按钮
   - 课程对学生可见

### 编辑现有课程

1. **修改课程信息**
   - 点击课程卡片的"编辑"按钮
   - 修改课程信息
   - 保存更改

2. **管理实验**
   - 点击"管理实验"按钮
   - 编辑现有实验或添加新实验
   - 调整实验排序
   - 删除不需要的实验

3. **控制发布状态**
   - 使用"发布/取消发布"按钮
   - 快速控制课程可见性

## 实验内容编写指南

### Markdown 格式示例

```markdown
# 实验目标

- 掌握 Linux 基本命令
- 理解文件系统结构
- 学会文件操作

## 实验环境

本实验使用 Ubuntu 22.04 系统。

## 实验步骤

### 1. 查看当前目录

使用 `pwd` 命令查看当前所在目录：

\`\`\`bash
pwd
\`\`\`

### 2. 列出文件

使用 `ls` 命令列出当前目录下的文件：

\`\`\`bash
ls -la
\`\`\`

### 3. 创建目录

创建一个名为 test 的目录：

\`\`\`bash
mkdir test
cd test
\`\`\`

### 4. 创建文件

创建并编辑文件：

\`\`\`bash
touch hello.txt
echo "Hello Spark Lab" > hello.txt
cat hello.txt
\`\`\`

## 判题标准

- 成功创建 test 目录（20分）
- 成功创建 hello.txt 文件（30分）
- 文件内容正确（50分）

## 提示

- 使用 Tab 键可以自动补全命令
- 使用上下箭头可以查看历史命令
- 遇到问题可以使用 `man` 命令查看帮助
```

### 内容编写建议

1. **结构清晰**
   - 使用标题层级组织内容
   - 步骤编号清晰
   - 代码块格式正确

2. **详细说明**
   - 每个步骤都有说明
   - 提供命令示例
   - 解释预期结果

3. **提供帮助**
   - 添加提示信息
   - 常见问题解答
   - 参考资料链接

## 容器配置建议

### 常用 Docker 镜像

| 镜像 | 用途 | 推荐配置 |
|------|------|----------|
| ubuntu:22.04 | Linux 基础 | CPU: 1.0, 内存: 512MB |
| python:3.11 | Python 编程 | CPU: 1.0, 内存: 512MB |
| node:18 | Node.js 开发 | CPU: 1.0, 内存: 512MB |
| mysql:8.0 | 数据库实验 | CPU: 2.0, 内存: 1024MB |
| nginx:latest | Web 服务器 | CPU: 0.5, 内存: 256MB |

### 资源配置原则

1. **CPU 限制**
   - 基础实验：0.5-1.0 核
   - 编程实验：1.0-2.0 核
   - 数据库/编译：2.0-4.0 核

2. **内存限制**
   - 命令行实验：256-512MB
   - 编程实验：512-1024MB
   - 数据库实验：1024-2048MB

3. **时间限制**
   - 简单操作：15-30 分钟
   - 编程任务：30-60 分钟
   - 综合项目：60-120 分钟

## 注意事项

### 删除操作
- 删除课程会同时删除所有实验
- 删除前会有确认提示
- 删除操作不可恢复

### 发布状态
- 未发布的课程学生不可见
- 已发布的课程立即对学生可见
- 可以随时切换发布状态

### 实验排序
- 排序号决定实验显示顺序
- 建议使用 1, 2, 3... 递增
- 可以使用 10, 20, 30... 便于插入

### 内容更新
- 修改已发布课程会立即生效
- 建议在低峰期更新
- 重要更新前先取消发布

## 最佳实践

### 课程设计
1. 由浅入深安排实验顺序
2. 每个实验专注一个知识点
3. 提供充足的练习机会
4. 设置合理的难度梯度

### 实验设计
1. 目标明确，步骤清晰
2. 提供完整的示例代码
3. 包含常见问题解答
4. 设置合理的时间限制

### 内容维护
1. 定期检查实验可用性
2. 更新过时的技术内容
3. 收集学生反馈改进
4. 保持内容与时俱进

## 快捷操作

### 快速发布课程
1. 创建课程（未发布）
2. 添加所有实验
3. 测试实验可用性
4. 点击"发布"按钮

### 批量管理
1. 使用排序号统一管理实验顺序
2. 复制现有实验内容快速创建新实验
3. 使用模板化的 Markdown 内容

### 测试流程
1. 创建测试课程
2. 添加实验并测试
3. 确认无误后复制到正式课程
4. 删除测试课程
