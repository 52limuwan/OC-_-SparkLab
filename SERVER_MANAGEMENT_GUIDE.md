# 服务器管理系统使用指南

## 概述

星火实验室现在支持**远程服务器管理**，管理员可以添加多台远程服务器，系统会自动在这些服务器上创建和管理 Docker 容器，实现分布式容器部署。

## 核心功能

### 1. 服务器管理
- 添加/删除远程服务器
- 支持 SSH 密码和私钥认证
- 实时监控服务器状态（CPU、内存、容器数）
- 测试服务器连接
- 自动负载均衡

### 2. 容器分配
- 自动选择负载最低的服务器
- 在远程服务器上创建 Docker 容器
- 支持容器的启动、停止、删除
- 远程执行命令

### 3. 监控统计
- 每分钟自动更新服务器状态
- CPU 和内存使用率监控
- 活跃容器数统计

## 使用步骤

### 第一步：准备远程服务器

在远程服务器上安装 Docker：

```bash
# Ubuntu/Debian
curl -fsSL https://get.docker.com -o get-docker.sh
sudo sh get-docker.sh

# 启动 Docker
sudo systemctl start docker
sudo systemctl enable docker

# 验证安装
docker --version
```

### 第二步：配置 SSH 访问

#### 方式 1：使用密码认证（简单）

确保远程服务器允许 SSH 密码登录：

```bash
# 编辑 SSH 配置
sudo nano /etc/ssh/sshd_config

# 确保以下配置
PasswordAuthentication yes

# 重启 SSH 服务
sudo systemctl restart sshd
```

#### 方式 2：使用私钥认证（推荐）

```bash
# 在本地生成密钥对
ssh-keygen -t rsa -b 4096

# 复制公钥到远程服务器
ssh-copy-id user@remote-server

# 获取私钥内容（用于添加服务器）
cat ~/.ssh/id_rsa
```

### 第三步：在管理后台添加服务器

1. 登录管理员账号
2. 进入 **服务器管理** 页面
3. 点击 **添加服务器**
4. 填写服务器信息：
   - 服务器名称：如 "实验服务器1"
   - 主机地址：IP 或域名
   - SSH 端口：默认 22
   - 用户名：通常是 root
   - 认证方式：选择密码或私钥
   - 资源配置：最大容器数、CPU 核心数、内存大小

5. 点击 **添加** 按钮
6. 系统会自动测试连接并检查 Docker 是否安装

### 第四步：验证服务器状态

添加成功后，服务器卡片会显示：
- 状态：在线/离线/错误
- 容器数：当前活跃容器 / 最大容器数
- CPU 使用率
- 内存使用率
- 资源配置

点击 **测试连接** 可以手动测试服务器连接。

### 第五步：学生使用

学生创建实验容器时，系统会：
1. 自动选择负载最低的在线服务器
2. 通过 SSH 在远程服务器上创建 Docker 容器
3. 返回容器访问信息（SSH、VNC、IDE 端口）

学生无需关心容器在哪台服务器上，系统自动管理。

## 技术实现

### 后端

1. **SSH 服务** (`SSHService`)
   - 使用 `node-ssh` 库建立 SSH 连接
   - 维护连接池，复用连接
   - 执行远程 Docker 命令

2. **服务器服务** (`ServerService`)
   - 管理服务器 CRUD
   - 负载均衡算法（选择容器数最少的服务器）
   - 定时任务更新服务器状态

3. **容器服务** (`ContainerService`)
   - 支持本地和远程容器创建
   - 自动选择最佳服务器
   - 统一的容器管理接口

### 数据库

新增 `Server` 模型：
- 服务器基本信息（名称、地址、端口）
- SSH 认证信息（密码/私钥）
- 资源配置（CPU、内存、最大容器数）
- 实时状态（CPU 使用率、内存使用率、活跃容器数）

`Container` 模型新增 `serverId` 字段，关联到服务器。

### 前端

新增 **服务器管理** 页面：
- 服务器列表展示
- 添加服务器表单
- 实时状态监控
- 测试连接功能

## 安全建议

1. **使用私钥认证**：比密码更安全
2. **限制 SSH 访问**：配置防火墙，只允许管理后台 IP 访问
3. **加密存储**：密码和私钥应该加密存储（TODO）
4. **定期更新**：及时更新服务器和 Docker 版本
5. **监控日志**：定期检查 SSH 和 Docker 日志

## 故障排查

### 服务器显示"错误"状态

1. 检查网络连接
2. 验证 SSH 端口是否开放
3. 确认用户名和密码/私钥正确
4. 查看后端日志

### 容器创建失败

1. 检查服务器 Docker 是否运行
2. 确认服务器资源充足
3. 验证镜像是否存在
4. 查看容器日志

### 连接超时

1. 增加 SSH 超时时间
2. 检查网络延迟
3. 确认防火墙规则

## API 接口

### 服务器管理

```typescript
// 获取所有服务器
GET /servers

// 获取单个服务器
GET /servers/:id

// 添加服务器
POST /servers
{
  name: string;
  host: string;
  port: number;
  username: string;
  authType: 'password' | 'key';
  password?: string;
  privateKey?: string;
  maxContainers?: number;
  cpuCores?: number;
  totalMemory?: number;
}

// 更新服务器
PUT /servers/:id

// 删除服务器
DELETE /servers/:id

// 测试连接
POST /servers/:id/test

// 连接服务器
POST /servers/:id/connect

// 断开连接
POST /servers/:id/disconnect
```

## 总结

通过这个服务器管理系统，你可以：
- 轻松管理多台远程服务器
- 自动分配容器到不同服务器
- 实时监控服务器状态
- 实现分布式容器部署

这样就不再是"容器管理"，而是真正的"服务器管理"了！
