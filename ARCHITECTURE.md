## 核心模块说明

### 1. 认证模块 (Auth Module)

- JWT Token 认证
- 密码 bcrypt 加密
- 用户注册/登录
- Token 自动刷新

### 2. Docker 管理模块 (Docker Module)

- 容器生命周期管理（创建/启动/停止/删除）
- 资源限制配置
- 用户与容器绑定
- 容器状态监控

### 3. 终端模块 (Terminal Module)

- WebSocket 实时通信
- PTY (伪终端) 转发
- 终端大小自适应
- 多会话管理

### 4. 实验室模块 (Lab Module)

- Markdown 题目管理
- 实验进度跟踪
- 自动判题接口（预留）

## 数据流

### 用户登录流程

```
1. 用户输入账号密码
2. Frontend → POST /auth/login
3. Backend 验证密码
4. 返回 JWT Token
5. Frontend 存储 Token 到 localStorage
6. 后续请求携带 Token
```

### 容器创建流程

```
1. 用户点击"启动容器"
2. Frontend → POST /docker/container/create
3. Backend 检查用户是否已有容器
4. 调用 dockerode 创建容器
5. 配置资源限制
6. 启动容器
7. 返回 containerId
8. 更新用户记录
```

### 终端通信流程

```
1. Frontend 建立 WebSocket 连接
2. 发送 'start' 事件 + containerId
3. Backend 创建 exec 会话
4. 启动 /bin/bash
5. 监听 stdout/stderr
6. 用户输入 → 'input' 事件 → stdin
7. 容器输出 → 'output' 事件 → Frontend
8. xterm.js 渲染输出
```

## 安全设计

### 1. 认证安全

- JWT Token 有效期 24 小时
- 密码 bcrypt 加密（salt rounds: 10）
- 敏感接口需要 JWT 验证

### 2. 容器安全

- 非 root 用户运行（labuser）
- 资源限制防止滥用
- 容器隔离（独立网络命名空间）
- 禁止特权模式

### 3. 网络安全

- CORS 配置限制来源
- WebSocket 连接验证
- API 请求频率限制（可扩展）
